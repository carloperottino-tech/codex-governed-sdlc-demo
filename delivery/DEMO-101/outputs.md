# DEMO-101 output locations

The governed workflow must create the following files in this directory during
the live demo:

- `plan.md` before application code is edited;
- `adr.md` for material design decisions;
- `verification.md` after tests and security checks; and
- `release-evidence.json` after verification.

These files do not exist in the baseline. Their absence is intentional and
prevents placeholders from being mistaken for completed work or approval.
